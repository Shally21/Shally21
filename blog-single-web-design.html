What do i need to ask my client to design a website?

  • Olajide Shalom
  • Web Design
  • 14

Most Software engineer today knows the importance of software development processes which consist of various phases and each phases ending with a defined output.The primary reason for using a phased process is that it breaks the problem of developing software into successfully performing set of phases, each handling a different concern of software development.

Requirement Gathering:It involves conducting a preliminary meeting with the clients to understand and analyse his needs.

Feasibility Studies:A feasibility study is conducted to find out whether a proposed system or project will be possible to build with given technology and resources, affordability of the system given the time and constraints of the organization , and acceptable for use by the eventual user of the system.it provides some answer to troublesoe questions like: Can I undertake this project?, what are my market constraints?, how do i price my products?, what benefits will accrue from this project?

Pricing:Which is one of the most important but at the same time most unpredictable stage. Example building a simple website might cost $10/hr but because of the inflation of naira to dollar you might want to charge more so you will not be at huge loss.But, before this understand your customers requirement so you will not end up ruining the project.

Timeline:The time frame is the next after understanding and defining the problem.The developer should know his timing to give accuracy.

Identification and analysis of customer needs. System evaluation for feasibility, Performing economic and technical analysis, Allocating function to manpower,database,hardware,software and other system elements, Establishing cost and schedule constraints, Creating a system definition that forms a foundation for other development.

"Websites promote you 24/7:No employee will do that."

Apart from being a good programmer, a software engineer must be familiar with various design approaches to be able to translate user requirement into specifications. A software engineer following and unerstanding each of this steps that the users need should be able to design,develop and deploy according to the users specifications.
